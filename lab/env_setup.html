1. WSL2 のインストール

Windows Subsystem for Linux 2（WSL2）を使うと，Windows 上で Ubuntu 環境を動かせます． LAMMPS・OpenFOAM などのシミュレーションソフトは Linux ネイティブで動作するため，WSL2 の利用を推奨します．

1
管理者権限で PowerShell を起動し，以下を実行：
PowerShell（管理者）
wsl --install

再起動後，Ubuntu が自動的にインストールされます．ユーザー名とパスワードを設定してください．

2
WSL バージョンの確認：
PowerShell
wsl --list --verbose

VERSION が 2 になっていれば OK です．

3
Ubuntu の初期設定：
Ubuntu (WSL2)
sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential curl wget git vim unzip
ファイルシステムの注意： Windows のファイル（C:\Users\...）は WSL2 から /mnt/c/Users/... としてアクセスできますが， I/O が遅いため，シミュレーションファイルは WSL2 のホームディレクトリ（~/）に置くことを推奨します．

2. VSCode のインストール

VSCode（Visual Studio Code）は Microsoft 製の無料エディタです．WSL2 上のファイルを Windows GUI から編集できます．

  1. https://code.visualstudio.com/ からインストーラをダウンロード・実行
  2. インストール時に「PATH への追加」にチェックを入れる
  3. VSCode を起動し，左サイドバーの拡張機能アイコンから WSL 拡張機能をインストール
  4. WSL2 ターミナルで code . を実行すると VSCode が WSL 接続モードで起動します
Ubuntu (WSL2) — VSCode を WSL モードで起動
cd ~/myproject
code .
ポイント： code . で開いた VSCode は Linux ファイルシステムに直接アクセスするため， ファイル保存・実行が高速です．ターミナルも VSCode 内から WSL2 を使用します．

3. Python / Conda 環境

Miniforge のインストール（推奨）

Anaconda より軽量な Miniforge（conda-forge チャンネルベース）を推奨します．

Ubuntu (WSL2)
curl -L https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-Linux-x86_64.sh -o miniforge.sh
bash miniforge.sh -b -p ~/miniforge3
~/miniforge3/bin/conda init bash
source ~/.bashrc

研究用 conda 環境の作成

Ubuntu (WSL2)
# 基本的な計算科学環境
conda create -n sim python=3.11 -y
conda activate sim

# 基本パッケージ
conda install -y numpy scipy matplotlib pandas jupyter

# 分子シミュレーション関連
pip install ase          # Atomic Simulation Environment
pip install ovito        # 可視化・解析（CLIモード）

# 機械学習力場
pip install mace-torch   # MACE

環境の管理

よく使うコマンド
conda activate sim        # 環境を有効化
conda deactivate          # 環境を無効化
conda env list            # 環境一覧
conda list                # インストール済みパッケージ一覧
conda env export > env.yml  # 環境をエクスポート（共有用）
conda env create -f env.yml  # 環境をインポート

4. Git の設定

Ubuntu (WSL2)
# ユーザー情報の設定
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
git config --global core.editor "code --wait"

# 日本語ファイル名の文字化け防止
git config --global core.quotepath false

# デフォルトブランチを main に設定
git config --global init.defaultBranch main

基本的な Git ワークフロー

よく使うコマンド

git clone URL              # リモートからクローン
git status                 # 変更状態確認
git add file.py            # ステージング
git commit -m "message"    # コミット
git push origin main       # プッシュ
git pull                   # プル（最新取得）
git log --oneline          # 履歴確認

5. SSH 設定（計算サーバーへの接続）

SSH 鍵の生成

Ubuntu (WSL2)
# ED25519 鍵ペアを生成
ssh-keygen -t ed25519 -C "your@email.com"
# → ~/.ssh/id_ed25519 (秘密鍵) と ~/.ssh/id_ed25519.pub (公開鍵) が生成される

# 公開鍵をサーバーに登録
ssh-copy-id -i ~/.ssh/id_ed25519.pub username@server.address

~/.ssh/config の設定（接続の簡略化）

~/.ssh/config
Host calc-server
    HostName  server.address.ac.jp
    User      your_username
    IdentityFile ~/.ssh/id_ed25519
    ServerAliveInterval 60

設定後は ssh calc-server だけで接続できます．

VSCode Remote-SSH： VSCode の Remote-SSH 拡張機能を使うと，計算サーバー上のファイルを VSCode GUI で直接編集できます． 接続先の ~/.ssh/config に登録されたホスト名が自動的に候補として表示されます．

ファイル転送

ファイル転送コマンド
# ローカル → サーバー
scp local_file.txt calc-server:~/remote_dir/

# サーバー → ローカル
scp calc-server:~/result.txt ./

# ディレクトリごと転送（rsync 推奨）
rsync -avz ./local_dir/ calc-server:~/remote_dir/

7. トラブルシューティング

症状原因・解決策
wsl --install が失敗する BIOS で仮想化（Intel VT-x / AMD-V）が有効か確認．Windows の「Windows の機能の有効化または無効化」から「仮想マシン プラットフォーム」と「Linux 用 Windows サブシステム」を有効にする
WSL2 から GPU が使えない NVIDIA の WSL2 対応ドライバーをインストール（nvidia-smi で確認）
code . を実行しても VSCode が開かない VSCode の Remote - WSL 拡張機能をインストール後，PATH を確認．export PATH="$PATH:/mnt/c/Users/username/AppData/Local/Programs/Microsoft VS Code/bin" を ~/.bashrc に追記
conda activate が効かない source ~/.bashrc を実行後，再度試す
SSH 接続でパスワードを求められる ~/.ssh/ のパーミッション確認：chmod 700 ~/.ssh && chmod 600 ~/.ssh/id_ed25519
