1. LAMMPS とは

LAMMPS（Large-scale Atomic/Molecular Massively Parallel Simulator）は Sandia 国立研究所が開発したオープンソースの古典 MD シミュレーションソフトウェアです． 多様な力場・アンサンブル・解析手法をサポートし，大規模並列計算（MPI）に対応しています．

特徴内容
ライセンスGPL（オープンソース）
並列化MPI + OpenMP + GPU (KOKKOS)
対応力場LJ, EAM, ReaxFF, MACE, SNAP, GAP など多数
公式ドキュメントdocs.lammps.org

2. インストール

Ubuntu (WSL2) へのインストール

Ubuntu (WSL2)
# conda からインストール（簡単）
conda install -c conda-forge lammps

# または apt から（バージョンが古い場合あり）
sudo apt install lammps

# 動作確認
lammps -h
ソースビルド：MACE・KOKKOS（GPU）など特定パッケージが必要な場合はソースからのビルドが必要です． 公式ドキュメントの "Build LAMMPS" セクションを参照してください．

3. 入力ファイルの構成

LAMMPS の入力ファイル（in.xxx）はコマンドを上から順番に実行します．

simulation/ ├── in.lj ← メイン入力ファイル ├── data.lj ← 原子座標・結合情報 ├── log.lammps ← 実行ログ（自動生成） ├── dump.lammpstrj ← トラジェクトリ（dump コマンドで出力） └── thermo.dat ← 熱力学量（fix print で出力）

入力ファイルのセクション構成

in.lj — 入力ファイルの基本構造
# ── 1. 初期設定 ──────────────────────────────────
units        lj          # 換算単位
dimension    3
atom_style   atomic

# ── 2. シミュレーションボックスと原子配置 ──────────
lattice      fcc 0.8442  # FCC, 密度 ρ*=0.8442
region       box block 0 5 0 5 0 5
create_box   1 box
create_atoms 1 box

# ── 3. 力場の設定 ────────────────────────────────
pair_style   lj/cut 2.5
pair_coeff   1 1 1.0 1.0 2.5   # ε σ rc (換算単位)

# ── 4. 速度の初期化 ──────────────────────────────
velocity     all create 1.44 87287 loop geom

# ── 5. アンサンブルと時間積分 ────────────────────
fix          1 all nve
fix          2 all temp/rescale 100 1.44 1.44 0.02 1.0

# ── 6. 出力設定 ──────────────────────────────────
thermo_style custom step temp pe ke etotal press
thermo       100
dump         1 all atom 500 dump.lammpstrj

# ── 7. 実行 ──────────────────────────────────────
timestep     0.005
run          10000

4. LJ 系のチュートリアル（完全版）

in.lj_full — LJ 液体（NVT, 平衡化 + 本計算）
# LJ 液体シミュレーション（換算単位）
# 条件: N=500, ρ*=0.8, T*=1.0

units        lj
atom_style   atomic
boundary     p p p     # 全方向周期境界

# 初期構造
lattice      fcc 0.8
region       simbox block 0 5 0 5 0 5
create_box   1 simbox
create_atoms 1 box

mass         1 1.0

# LJ ポテンシャル（rc=2.5σ, シフト補正あり）
pair_style   lj/cut 2.5
pair_coeff   1 1 1.0 1.0
pair_modify  shift yes   # rc でポテンシャルをシフト

# 速度初期化 (T*=1.0)
velocity     all create 1.0 12345 dist gaussian

# ── 平衡化フェーズ (NVT) ─────────────────────────
fix          NVT all nvt temp 1.0 1.0 0.5
thermo_style custom step temp pe ke etotal press vol
thermo       200
timestep     0.005
run          20000

# 平衡化後に速度を再初期化
velocity     all scale 1.0

# ── 本計算フェーズ (NVE) ─────────────────────────
unfix        NVT
fix          NVE all nve
dump         trj all custom 100 dump.lammpstrj id type x y z vx vy vz
dump_modify  trj sort id
thermo       100
run          50000

write_data   final.data   # 最終構造を保存

実行コマンド

Terminal
# シリアル実行
lammps -in in.lj_full

# MPI 並列実行（4コア）
mpirun -np 4 lammps -in in.lj_full

5. 水のシミュレーション（SPC/E モデル）

in.water_spce — SPC/E 水モデル
units        real          # Å, kcal/mol, fs
atom_style   full
boundary     p p p

read_data    water.data    # 座標・結合情報を外部ファイルから読み込み

# SPC/E 力場
pair_style   lj/cut/coul/long 9.0 9.0
pair_coeff   1 1 0.1553 3.166    # O-O: ε(kcal/mol), σ(Å)
pair_coeff   1 2 0.0    0.0      # O-H
pair_coeff   2 2 0.0    0.0      # H-H

kspace_style pppm 1.0e-4  # 長距離静電気（PPPM）
kspace_modify mesh 32 32 32 order 4

# 結合・角度（SHAKE で O-H 距離・H-O-H 角度を拘束）
bond_style   harmonic
angle_style  harmonic
fix          SHAKE all shake 1e-4 20 0 b 1 a 1

# NVT (T=300K)
velocity     all create 300.0 4928459
fix          NVT all nvt temp 300.0 300.0 100.0
timestep     2.0          # [fs] SHAKE 使用で 2 fs 可
thermo       500
run          500000        # 1 ns

6. 出力ファイルと解析

OVITO による可視化

Terminal — OVITO で dump ファイルを開く
ovito dump.lammpstrj    # GUI で可視化

Python による解析（MSD, RDF）

analyze_lammps.py
import numpy as np
from ase.io.lammpsrun import read_lammps_dump_text

# トラジェクトリの読み込み
with open("dump.lammpstrj") as f:
    traj = list(read_lammps_dump_text(f, index=slice(None)))

print(f"フレーム数: {len(traj)}, 原子数: {len(traj[0])}")

# MSD の計算
pos0 = traj[0].get_positions()
msd = [np.mean(np.sum((t.get_positions() - pos0)**2, axis=1))
       for t in traj]

import matplotlib.pyplot as plt
plt.plot(msd)
plt.xlabel("Frame"); plt.ylabel("MSD (Å²)")
plt.savefig("msd.png", dpi=150)
plt.show()
