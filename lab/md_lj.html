分子動力学法の基礎
— LJ ポテンシャルと MD シミュレーション —

熱流体力学研究室 内部資料 | 2026年4月

1. 分子動力学法とは

分子動力学（Molecular Dynamics, MD）法は，原子・分子をニュートン力学に従う古典粒子として扱い，その運動方程式を数値的に時間積分することで，系の時間発展を求める計算手法です．

MD シミュレーションから得られる粒子の軌跡（位置と速度の時系列）に統計力学を適用することで，圧力・温度・熱伝導率・拡散係数などの巨視的物性を計算できます．

本資料では，最も基本的なモデルである Lennard-Jones（LJ）ポテンシャルを用いた MD シミュレーションの理論と Python による実装を説明します．

MD の基本的な考え方：
粒子間に相互作用ポテンシャルを定義 → 粒子に働く力を計算 → 運動方程式を時間積分 → 位置・速度を更新 → 繰り返す

2. Lennard-Jones ポテンシャル

2.1 ポテンシャル関数

LJ ポテンシャルは，2粒子間の距離 $r$ の関数として次式で表されます：

$$u_\text{LJ}(r) = 4\varepsilon \left[ \left(\frac{\sigma}{r}\right)^{12} - \left(\frac{\sigma}{r}\right)^{6} \right]$$
（式 2.1）

ここで，$\varepsilon$ はポテンシャルの深さ（エネルギースケール），$\sigma$ は粒子径（長さスケール）です．

  • $r^{-12}$ 項：短距離斥力（パウリ排除則による電子雲の重なりに対応）
  • $r^{-6}$ 項：長距離引力（ロンドン分散力）
  • 最小値（ポテンシャルの底）は $r = 2^{1/6}\sigma \approx 1.122\,\sigma$ のとき $u = -\varepsilon$
パラメータ意味アルゴン（Ar）の例
$\varepsilon / k_\text{B}$ポテンシャル深さ / ボルツマン定数119.8 K
$\sigma$粒子径（衝突直径）3.405 Å
$m$粒子質量39.948 u (6.634 × 10⁻²⁶ kg)

2.2 LJ 力

粒子 $j$ が粒子 $i$ に及ぼす力は，ポテンシャルの勾配から求められます：

$$\mathbf{f}_{ij} = -\frac{du_\text{LJ}}{dr}\hat{\mathbf{r}}_{ij} = \frac{48\varepsilon}{r^2}\left[\left(\frac{\sigma}{r}\right)^{12} - \frac{1}{2}\left(\frac{\sigma}{r}\right)^{6}\right]\mathbf{r}_{ij}$$
（式 2.2）

ここで $\mathbf{r}_{ij} = \mathbf{r}_i - \mathbf{r}_j$ は粒子 $j$ から粒子 $i$ へ向かうベクトル，$r = |\mathbf{r}_{ij}|$ です． なお，$r^2$ で割った形（$\mathbf{r}_{ij}$ とそのまま掛ける）で表すことで，$\sqrt{}$ の計算を一度省けます．

2.3 無次元化（換算単位）

MD シミュレーションでは，$\varepsilon$，$\sigma$，$m$ を基本単位とする換算単位（reduced units）を使うのが一般的です．これにより数値が $O(1)$ となり，数値計算が安定します．

物理量換算単位記号
長さ$\sigma$$r^* = r / \sigma$
エネルギー$\varepsilon$$u^* = u / \varepsilon$
質量$m$$m^* = m / m = 1$
時間$\tau = \sigma\sqrt{m/\varepsilon}$$t^* = t / \tau$
温度$\varepsilon / k_\text{B}$$T^* = k_\text{B} T / \varepsilon$
圧力$\varepsilon / \sigma^3$$P^* = P\sigma^3 / \varepsilon$
力$\varepsilon / \sigma$$f^* = f\sigma / \varepsilon$

換算単位での LJ ポテンシャルと力は次式となります（上付き $*$ を省略）：

$$u(r) = 4\left[\frac{1}{r^{12}} - \frac{1}{r^{6}}\right]$$ $$\mathbf{f}_{ij} = \frac{48}{r^2}\left[\frac{1}{r^{12}} - \frac{1}{2r^{6}}\right]\mathbf{r}_{ij}$$
（換算単位，式 2.3）

3. 運動方程式と数値積分

3.1 ニュートンの運動方程式

粒子 $i$ の位置 $\mathbf{r}_i$，速度 $\mathbf{v}_i$，質量 $m_i$ に対して，

$$m_i \ddot{\mathbf{r}}_i = \mathbf{F}_i = \sum_{j \neq i} \mathbf{f}_{ij}$$
（式 3.1）

$\mathbf{F}_i$ は粒子 $i$ に働く合力です．換算単位では $m_i = 1$ なので $\ddot{\mathbf{r}}_i = \mathbf{F}_i$ となります．

3.2 Velocity Verlet 法

時間ステップ $\Delta t$ での数値積分には Velocity Verlet 法が広く使われます．エネルギー保存性に優れ，時間反転対称性を持つシンプレクティック積分です．

$$\mathbf{r}_i(t+\Delta t) = \mathbf{r}_i(t) + \mathbf{v}_i(t)\,\Delta t + \frac{1}{2}\mathbf{a}_i(t)\,\Delta t^2$$ $$\mathbf{v}_i\!\left(t+\frac{\Delta t}{2}\right) = \mathbf{v}_i(t) + \frac{1}{2}\mathbf{a}_i(t)\,\Delta t$$ $$\mathbf{a}_i(t+\Delta t) = \frac{\mathbf{F}_i(t+\Delta t)}{m_i}$$ $$\mathbf{v}_i(t+\Delta t) = \mathbf{v}_i\!\left(t+\frac{\Delta t}{2}\right) + \frac{1}{2}\mathbf{a}_i(t+\Delta t)\,\Delta t$$
（式 3.2）
時間刻み $\Delta t$ の選び方： LJ 系の換算単位では $\Delta t^* \approx 0.001 \sim 0.005$ が一般的です． 小さすぎると計算コスト大，大きすぎるとエネルギーが保存しません． 目安として，系の特性時間（振動周期）の 1/100 以下を選びます．

4. 周期境界条件（PBC）

実際の系は $10^{23}$ 個程度の分子を含みますが，MD では高々数千〜数百万個しか扱えません． 表面効果を除くために周期境界条件（Periodic Boundary Conditions, PBC）を用います．

一辺 $L$ の立方体シミュレーションボックスを仮定すると，ある粒子が端から出ると，反対側から入ってきます：

$$r_{i,\alpha} \leftarrow r_{i,\alpha} - L \cdot \text{round}\!\left(\frac{r_{i,\alpha}}{L}\right)$$
（式 4.1，$\alpha = x, y, z$）

粒子間距離の計算には最近接像規約（minimum image convention）を使います． 粒子 $j$ として，ボックスの周期像のうち最も近いものを選びます：

$$\delta r_\alpha = r_{i,\alpha} - r_{j,\alpha} - L \cdot \text{round}\!\left(\frac{r_{i,\alpha} - r_{j,\alpha}}{L}\right)$$
（式 4.2）

5. カットオフ

$N$ 粒子系の全ペア数は $N(N-1)/2$ であり，$N=1000$ でも 50 万ペアに達します． LJ ポテンシャルは距離とともに急速に減衰するため，カットオフ距離 $r_c$ 以上の相互作用を無視します．

$$u(r) = \begin{cases} 4\left[\dfrac{1}{r^{12}} - \dfrac{1}{r^{6}}\right] - u(r_c) & (r \leq r_c) \\ 0 & (r > r_c) \end{cases}$$
（式 5.1，シフトカットオフ）

$u(r_c)$ を引くことでポテンシャルが $r_c$ で連続になります（シフト法）． 一般に $r_c = 2.5\,\sigma$（換算単位）が使われます． カットオフを使うことで計算量を $O(N^2)$ から，セルリストやバーレット法を併用すれば $O(N)$ に近づけられます．

注意：カットオフの影響（切断誤差）を補正するために，長距離補正（tail correction）を圧力・エネルギーに適用することがあります． $r_c = 2.5\,\sigma$ のとき，LJ 系への切断誤差は比較的小さいです．

6. 温度とエネルギー

運動エネルギーと温度はエネルギー等分配則より，

$$E_k = \frac{1}{2}\sum_{i=1}^{N} m_i |\mathbf{v}_i|^2 = \frac{3}{2} N k_\text{B} T$$
（式 6.1）

換算単位（$m=1$, $k_\text{B}=1$）では：

$$T^* = \frac{2 E_k}{3N}$$
（式 6.2）

ポテンシャルエネルギー：

$$E_p = \sum_{i < j} u_\text{LJ}(r_{ij})$$
（式 6.3）

速度スケーリング法（温度制御）： 平衡化の初期段階では，各ステップで速度を目標温度 $T_\text{target}$ にリスケールします：

$$\lambda = \sqrt{\frac{T_\text{target}}{T_\text{current}}}，\quad \mathbf{v}_i \leftarrow \lambda\,\mathbf{v}_i$$
（式 6.4）

7. MD アルゴリズムの流れ

基本的な MD ループ（Velocity Verlet + PBC + カットオフ）
1
初期化：粒子をFCC格子などに配置，Maxwell–Boltzmann 分布に従い速度を初期化（重心速度を 0 に補正）
2
力の計算：全ペアに対してカットオフ内のみ LJ 力を計算，PBC（最近接像規約）を適用
3
位置の更新（Velocity Verlet step 1）：$\mathbf{r}(t+\Delta t)$ と $\mathbf{v}(t+\Delta t/2)$ を計算
4
PBC の適用：ボックス外に出た粒子をボックス内に折り返す
5
力の再計算：新しい位置で全ペアの力を計算
6
速度の完全更新（Velocity Verlet step 2）：$\mathbf{v}(t+\Delta t)$ を計算
7
物理量の計算：運動エネルギー・温度・ポテンシャルエネルギー・圧力などを集計
8
温度制御（平衡化フェーズのみ）：速度スケーリングにより温度を目標値に近づける
9
ステップ 3 に戻る（指定ステップ数まで繰り返す）

8. Python プログラム例

以下は NumPy を使った LJ-MD シミュレーションの最小実装例です． 換算単位を使用し，3次元 $N$ 粒子系をシミュレートします． 教育目的の実装のため，力の計算は $O(N^2)$ の素直な実装です．

md_lj.py — LJ ポテンシャル MD シミュレーション（換算単位） 
"""
md_lj.py  —  Lennard-Jones 分子動力学シミュレーション（換算単位）
教育用実装：NumPy + Velocity Verlet + PBC + カットオフ
"""
import numpy as np
import matplotlib.pyplot as plt

# ─── パラメータ設定 ──────────────────────────────────────
N         = 108          # 粒子数（FCC: 4*n^3，n=3 → 108）
rho       = 0.8          # 数密度 ρ* = N/V  （換算単位）
T_target  = 1.0          # 目標温度 T*
dt        = 0.005        # 時間刻み Δt*
n_equil   = 2000         # 平衡化ステップ数
n_prod    = 5000         # 本計算ステップ数
r_cut     = 2.5          # カットオフ距離 rc*
save_interval = 100      # 物理量の記録間隔

# ─── ボックスサイズ ──────────────────────────────────────
L = (N / rho) ** (1.0 / 3.0)

# ─── カットオフでのポテンシャル値（シフト用） ──────────────
u_cut = 4.0 * (r_cut**-12 - r_cut**-6)


def init_fcc(n_cell, L):
    """FCC 格子に粒子を配置する"""
    basis = np.array([
        [0.0, 0.0, 0.0], [0.5, 0.5, 0.0],
        [0.5, 0.0, 0.5], [0.0, 0.5, 0.5],
    ])
    a = L / n_cell
    pos = []
    for ix in range(n_cell):
        for iy in range(n_cell):
            for iz in range(n_cell):
                for b in basis:
                    pos.append((np.array([ix, iy, iz]) + b) * a)
    return np.array(pos[:N])   # N 粒子分だけ返す


def init_velocities(N, T):
    """Maxwell–Boltzmann 分布で速度初期化，重心速度を 0 に補正"""
    vel = np.random.randn(N, 3)          # 標準正規分布
    vel -= vel.mean(axis=0)               # 重心速度を除去
    T_current = np.sum(vel**2) / (3 * N)
    vel *= np.sqrt(T / T_current)          # 目標温度にスケール
    return vel


def apply_pbc(pos, L):
    """周期境界条件：粒子をボックス内に折り返す"""
    pos -= L * np.floor(pos / L)
    return pos


def calc_forces(pos, L, r_cut, u_cut):
    """
    LJ ポテンシャルによる力とポテンシャルエネルギーを計算
    Returns: forces (N,3), potential_energy (float)
    """
    N = len(pos)
    forces = np.zeros((N, 3))
    pe = 0.0
    r_cut2 = r_cut**2

    for i in range(N - 1):
        for j in range(i + 1, N):
            # 最近接像規約
            dr = pos[i] - pos[j]
            dr -= L * np.round(dr / L)
            r2 = np.dot(dr, dr)

            if r2 < r_cut2:
                r2i  = 1.0 / r2
                r6i  = r2i ** 3
                r12i = r6i ** 2
                # ポテンシャルエネルギー（シフト付き）
                pe += 4.0 * (r12i - r6i) - u_cut
                # 力（f_ij = 48*(r12i - 0.5*r6i)*r2i * dr）
                f_mag = 48.0 * (2.0 * r12i - r6i) * r2i
                f_vec = f_mag * dr
                forces[i] += f_vec
                forces[j] -= f_vec    # ニュートン第3法則

    return forces, pe


def calc_temperature(vel):
    """運動エネルギーから温度を計算（換算単位）"""
    ke = 0.5 * np.sum(vel**2)
    T = 2.0 * ke / (3.0 * len(vel))
    return ke, T


# ─── 初期化 ──────────────────────────────────────────────
np.random.seed(42)
n_cell = round(N ** (1.0/3.0) / 4**(1.0/3.0))   # FCC セル数
pos = init_fcc(n_cell, L)
vel = init_velocities(N, T_target)
forces, pe = calc_forces(pos, L, r_cut, u_cut)

# ─── 記録用リスト ─────────────────────────────────────────
t_list, T_list, Ek_list, Ep_list, Etot_list = [], [], [], [], []

print(f"N={N}, L={L:.3f}, rho={rho}, T_target={T_target}")
print(f"dt={dt}, n_equil={n_equil}, n_prod={n_prod}")
print("Step        T*      Ek*     Ep*    Etot*")

# ─── 平衡化ループ ─────────────────────────────────────────
for step in range(n_equil):
    # Velocity Verlet step 1
    vel_half = vel + 0.5 * forces * dt
    pos += vel_half * dt
    pos = apply_pbc(pos, L)

    # 力の再計算
    forces, pe = calc_forces(pos, L, r_cut, u_cut)

    # Velocity Verlet step 2
    vel = vel_half + 0.5 * forces * dt

    # 速度スケーリング（温度制御）
    ke, T_cur = calc_temperature(vel)
    vel *= np.sqrt(T_target / T_cur)

    if step % save_interval == 0:
        ke, T_cur = calc_temperature(vel)
        print(f"[EQ] {step:5d}  {T_cur:.4f}  {ke/N:.4f}  {pe/N:.4f}  {(ke+pe)/N:.4f}")

# ─── 本計算ループ ─────────────────────────────────────────
for step in range(n_prod):
    # Velocity Verlet step 1
    vel_half = vel + 0.5 * forces * dt
    pos += vel_half * dt
    pos = apply_pbc(pos, L)

    # 力の再計算
    forces, pe = calc_forces(pos, L, r_cut, u_cut)

    # Velocity Verlet step 2
    vel = vel_half + 0.5 * forces * dt

    # 物理量の記録
    if step % save_interval == 0:
        ke, T_cur = calc_temperature(vel)
        etot = ke + pe
        t_list.append(step * dt)
        T_list.append(T_cur)
        Ek_list.append(ke / N)
        Ep_list.append(pe / N)
        Etot_list.append(etot / N)
        print(f"[MD] {step:5d}  {T_cur:.4f}  {ke/N:.4f}  {pe/N:.4f}  {etot/N:.4f}")

# ─── 結果のプロット ───────────────────────────────────────
fig, axes = plt.subplots(3, 1, figsize=(8, 9), sharex=True)

axes[0].plot(t_list, T_list, color="tab:red")
axes[0].axhline(T_target, ls="--", color="gray")
axes[0].set_ylabel("Temperature T*")

axes[1].plot(t_list, Ep_list, label="Potential", color="tab:blue")
axes[1].plot(t_list, Ek_list, label="Kinetic", color="tab:orange")
axes[1].legend()
axes[1].set_ylabel("Energy per particle E*/N")

axes[2].plot(t_list, Etot_list, color="tab:green")
axes[2].set_ylabel("Total Energy E*_tot/N")
axes[2].set_xlabel("Time t*")

for ax in axes:
    ax.grid(True, alpha=0.3)

fig.suptitle(f"LJ-MD: N={N}, ρ*={rho}, T*_target={T_target}")
plt.tight_layout()
plt.savefig("md_lj_result.png", dpi=150)
plt.show()
print("Done. → md_lj_result.png")

NumPy ベクトル化版：力の計算（高速版）

上のコードの calc_forces 関数を以下に置き換えると NumPy の配列演算で高速化できます：

NumPy ベクトル化版 calc_forces（O(N²) だが高速） 
def calc_forces(pos, L, r_cut, u_cut):
    """NumPy ベクトル化版：全ペアを一括計算"""
    N = len(pos)
    # 全ペアの変位ベクトル (N, N, 3)
    dr = pos[:, np.newaxis, :] - pos[np.newaxis, :, :]
    # 最近接像規約
    dr -= L * np.round(dr / L)
    # 距離の二乗 (N, N)
    r2 = np.sum(dr**2, axis=-1)
    np.fill_diagonal(r2, np.inf)   # 自己相互作用を除外

    mask = r2 < r_cut**2
    r2_masked = np.where(mask, r2, 1.0)   # 0除算防止
    r2i  = 1.0 / r2_masked
    r6i  = r2i ** 3
    r12i = r6i ** 2

    # ポテンシャルエネルギー（上三角のみ和を取って二重計算を避ける）
    ij_mask = mask & (np.arange(N)[:, None] < np.arange(N)[None, :])
    pe = np.sum((4.0 * (r12i - r6i) - u_cut) * ij_mask)

    # 力 f_ij = 48*(2*r12i - r6i)*r2i * dr  (antisymmetric)
    f_scalar = np.where(mask, 48.0 * (2.0 * r12i - r6i) * r2i, 0.0)
    forces = np.sum(f_scalar[:, :, np.newaxis] * dr, axis=1)

    return forces, pe
